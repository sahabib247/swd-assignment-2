VAR:

  • Function-scoped variables, declared by 'var', are only visible inside the function in which they are declared. Global variables are those that are declared outside of any function.

  • 'var' variables are hoisted, which means that during compilation, the declaration is moved to the top of the function or global context. The initialization is still in effect, though.

LET:

  • 'let'-declared variables are block-scoped, which means you may access them only inside the block (two curly brackets) in which they are specified.

  • Let variables are not hoisted in the same manner as var. The interpreter gets to the declaration before the variable is initialized, even though they are hoisted.

CONST:

  • Variables that are declared with 'const' are also block-scoped.

  • When using 'const' with objects or arrays, the reference is also a constant, but the properties or elements inside the object or array can still be modified.

In JavaScript, 'null' and 'undefined' are special values that represent the absence of a meaningful value.

NULL:

  • Null is typically used as an intentional absence of any object value.

  • It can be assigned to a variable to indicate that it has no meaningful value or that it has been explicitly set to nothing.

UNDEFINED:

  • It indicates that a variable has been declared but has not been assigned a value.

  • Variables which have not been initialized are automatically assigned to 'undefined'.

REST API stands for Representational State Transfer Application Programming Interface. It is an architectural style for designing networked applications. It indicates that a variable has been declared but has not been assigned a value.

REST API provides a set of rules for building web services that are scalable, stateless, can easily be consumed by various clients, which makes it a popular choice for web development.
